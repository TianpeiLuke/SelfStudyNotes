data categoies


For the purpose of COAP, we requires no special categories of data that associated with customer's real identities to be included in the system since they are not related to the definition of buyer abuses Based on the available CS transcript data, we provide the following guidance on how we use them, collect them and store them. 



| **Data Categories** | **Data Elements** | **Description** | **Sensitive Personal Information** | **Usage** | **Collect** | **Storage** |
| ---- | ---- | ---- | ---- | ---- | ---- | ---- |
| Unique Identifiers | **contact_id** | unique identifier for the contact | **No** | To index the sample | **Digest to API call to** [Junction service](https://w.amazon.com/bin/view/CS_Neuron_and_Wit/Neuron/Junction/) | **EDX with 365 days retention** |
| **customer_id** | unique identifier for customer (buyer) | **No** | The key for customer-level investigation |  |  |  |
| **marketplace_id** | the unique marketplace identifier for the one that the contact belongs to | **No** | **The key to associate with marketplace-level policies** |  |  |  |
| **customer_order_ids** | the order IDs associated with the contact | **No** | **The key to associate with order-level investigation** |  |  |  |
| **asins** | the unique identifiers of the products (in the orders) that the customer is contacting about | **No** | **The key to associate with asin-level policies** |  |  |  |
| **skill_id** | unique identifier for a grouping that the agent falls into and is representative of the contacts that they are trained to handle. | **No** | **This information is useful to rank the trustworthiness of the CS contact** |  |  |  |
| For Amazon Internal Usage | **sic_codes** | the agent selected SIC codes when wrapping up the contact. | **No** | **This is likely useful to filter out irrelevant contacts** |  |  |
| Timestamp | **creation_date** | the epoch time the contact was created | **No** | **To define time window for retrieval** |  |  |
| Customer Contact Information | **raw_transcript** | contains the raw chat transcript retrieved from CSCaseQueryService (the authoritative source). | **Yes.** | As input to associated machine learning models in order to identify if there exists strong evidences that suggest the buyer has violated Amazon's refund and return polices |  |  |
|  | **agent_annotations** | the raw annotations (free text) entered by the customer service agent handling the contact | **Yes** | This information is used as side note to filter out contacts that are not related to refund/return events. |  |  |
|  | **auto_annotations** | the CS Central added annotations added automatically when certain actions are taken | **Yes** | We may not digest this field |  |  |
